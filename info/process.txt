- GPT 5.1 Thinking을 활용해 여러 Agent가 순차적으로 작업하여 PPT (를 생성할 수 있는 HTML 코드)를 생성하는 프롬프트를 테스트 함

 - 사용자가 어떤 PPT를 만들려고 하는지 알 수 없기 때문에 General하게 모든 요청에 대응할 수 있는 구조를 고민해 봄

 - 한번의 요청에 특화된 작업만 처리하는 프롬프트를 만드는게 더 나은 성능을 보이므로 이 Agent (프롬프트)를 순차적으로 엮어서 최종 산출물을 만들 수 있는 구조로 진행

단계는 1. 자료 조사 → 2. 기본 리포트 생성 → 3. HTML 코드 생성 → HTML 코드 검증 및 수정 의 4 단계로 나눔
각 단계의 프롬프트는 사용자의 다양한 리퀘스트를 받을 수 있도록 세 부분으로 나눠 구조화 되어 있음
Role : 단계별 프롬프트가 진행해야 하는 역할을 상세하게 설명하는 부분
Request : 사용자가 입력한 텍스트 기반 명령을 입력하는 부분
Resource : 이전 단계의 결과를 입력하는 부분
